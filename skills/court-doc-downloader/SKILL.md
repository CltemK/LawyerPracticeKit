# court-doc-downloader

## Description

法院送达文书下载工具（v8）。从 zxfw.court.gov.cn 下载所有送达 PDF，或处理微信等渠道传来的传票图片/扫描件，自动进行文字提取与 OCR 识别，根据文书类型自动命名，**同时保存到本地指定目录**和上传至飞书云空间（按案号归类），并为传票创建飞书团队日历开庭日程。每个案号文件夹内会上传一份 `案件信息.txt`，包含识别的案件信息和供用户编辑的我方/对方当事人字段。

## Use Cases

用户发送包含法院送达链接的消息**或上传传票图片/PDF附件**时使用。完整流程：获取文件列表 → 按案号分组 → 逐个下载到临时目录 → PDF 文字提取/OCR 识别 → 文书识别 → 自动命名 → 上传飞书云空间 →（传票则创建飞书日历）。

**支持的输入类型：**
- 法院送达链接（zxfw.court.gov.cn）
- 微信/其他渠道传来的传票图片（.jpg/.png）或 PDF 扫描件（均视为传票处理）

## Execution

```bash
# 法院送达链接（默认流程）
python court_doc_downloader.py <URL>

# 处理本地上传的传票图片/PDF（识别后按文书类型处理）
python court_doc_downloader.py --local-file <文件路径>

# 指定飞书父文件夹（传入 folder_token）
python court_doc_downloader.py <URL> --parent-folder <folder_token>

# 跳过飞书日历创建
python court_doc_downloader.py <URL> --skip-calendar

# 只下载指定文件
python court_doc_downloader.py <URL> --files "1,3,5"
```

## Output

- **同时保存到本地和上传飞书**：下载到本地指定目录，同时上传至飞书云空间（上传后保留本地副本）
- **本地保存目录**：`D:/多平台同步文件/05诉讼项目/11我的法院送达文件`（可由 `COURT_OUTPUT_DIR` 环境变量覆盖）
- **飞书文件夹命名规则**：`{案号}_{最新送达日期}`（如 `（2026）鄂0103民初1234号_20260411`）
  - 案号：从文书名称（c_wsmc）中提取；无法提取时标记为"未知案号"
  - 最新送达日期：该案号下所有文件中最早的 `dt_cjsj`（送达文件创建时间），格式 YYYYMMDD
  - 同一案号的多份文件（如传票+判决书+通知书）全部归入同一文件夹
- **同名文件自动加序号避免覆盖**
- **传票**重命名: `开庭日期_法院名称_传票.pdf`（如 `2026年04月16日14:50_武汉市江汉区人民法院_传票.pdf`）
- **判决书**重命名: `原告与被告+案由+纠纷_读取日期.pdf`
- **其他文书**（须知/通知/告知书等）：使用 API 返回的文书名称，含案号的文书加案号后缀区分
- **传票**自动在飞书**团队日历**中创建开庭日程
- **每个飞书文件夹**内同时上传 `案件信息.txt`，记录案件摘要（含供用户编辑的当事人字段）

## 处理流程

1. **获取文件列表**：调用法院 API `getWsListBySdbhNew` 获取所有送达文书（支持多文件）
2. **按案号分组**：
   - 从每份文书的名称（c_wsmc）中提取案号
   - 无法提取时该文件独立成组（以文书名称作为标识）
   - 记录每个案号组中最新的 `dt_cjsj`（送达时间）
3. **创建飞书文件夹**：
   - 检查目标飞书文件夹中是否已有同名文件夹（有则复用）
   - 无则创建，文件夹名称为 `{案号}_{最新送达日期}`
   - 支持指定 `--parent-folder` 上传到指定文件夹，否则上传到云空间根目录
4. **逐个下载**：通过 PDF 直链（含签名）下载到临时目录，同名文件自动加序号避免覆盖
5. **文字提取**：
   - 优先用 `pdfplumber` 提取文本（文本型 PDF）
   - 文字不足时调用 `mineru-extract` skill 的 `mineru_parse_documents.py` 对 PDF 直链 URL 做 OCR（扫描型 PDF）
6. **文书识别**：根据内容关键词判断类型（传票 / 判决书 / 其他）
7. **自动命名**：
   - 传票：提取"开庭日期"+"法院名称" → `2026年04月16日14:50_武汉市江汉区人民法院_传票.pdf`
   - 判决书：提取"我所代理的原告"+"被告"+"案由" → `张三与李四民间借贷纠纷_2026年01月15日.pdf`
   - 其他：使用原始文书名称，有案号的加案号后缀
8. **上传飞书**：重命名后立即上传至对应案号的飞书文件夹，上传成功后清理本地临时文件
9. **生成案件信息 TXT**：为每个案号文件夹生成 `案件信息.txt`，记录案号、我方当事人（空供用户填写）、对方当事人（空供用户填写）、案由、开庭时间、送达记录列表，上传到飞书同文件夹
10. **飞书团队日历**（仅传票）：调用 `lark-cli calendar events create` 创建开庭日程

## 文书类型识别规则

### 判决书识别（三条必须同时满足）
以下三条**全部满足**才判定为判决书；缺少任意一条则不认定为判决书：
1. PDF 第一页包含"人民法院判决书"字样
2. 内容包含原告/被告 **或** 上诉人/被上诉人
3. 内容包含案号（如 `（2026）鄂0103民初1234号`）

### 其他类型
- 标题/内容含"传票" → **传票**
- 含"须知/通知/指南/告知书/权利义务" → **其他**
- 以上均不满足时：按关键词评分（传票关键词 vs 判决书关键词）

## 判决书解析规则（重要）

### 文书处理原则
- **撤诉裁定书**：不需要解析，内容无实质性
- **判决书**：需要解析，重点在"本院认为"部分

### 解析目标
解析判决书时，提取以下结构化信息：
1. **争议焦点**：当事人核心争议是什么
2. **裁判观点**：法院对本案的实质性判断（重点）
3. **裁判结果**：判决结论（支持/驳回/部分支持）

### 一审判决书解析方法
- 本院认为**只出现一次**
- "本院认为"后面的内容就是裁判观点
- 格式：`本院认为，...` → 提取冒号或逗号后的整段

### 二审判决书解析方法
- 本院认为**通常出现两次**
  - "一审法院认为"：一审法院的观点（可忽略）
  - "本院认为"：二审判决法院的观点（**重点**）
- **只需总结二审法院的"本院认为"部分**
- 格式：`本院认为，...` → 提取冒号或逗号后的整段

### 解析示例
```
一审："...本院认为，被告未按约定履行合同义务..."
  → 裁判观点：被告未按约定履行合同义务

二审："...一审法院认为，原告主张的损失与被告无因果关系..."
  → 忽略一审观点
  "...本院认为，原告提交的证据足以证明损失存在..."
  → 裁判观点：原告提交的证据足以证明损失存在
```

### 注意事项
- 撤诉裁定书直接跳过，不做实质性解析
- 二审优先看"本院认为"，忽略"一审法院认为"
- 裁判观点的核心在本院认为之后的内容

## 环境变量

| 变量 | 用途 | 默认值 |
|------|------|--------|
| `COURT_FEISHU_FOLDER_TOKEN` | 飞书上传目标父文件夹 token | 空（根目录）|
| `COURT_TEAM_CALENDAR_ID` | 飞书团队日历 ID（用于创建开庭日程） | 空（必填）|
| `COURT_OUTPUT_DIR` | 本地保存目录（同时上传到飞书） | `D:/多平台同步文件/05诉讼项目/11我的法院送达文件` |
| `LARK_CLI_PATH` | lark-cli 可执行文件路径（可选） | 自动查找 |
| `MINERU_EXTRACT_SCRIPT` | MinerU OCR 脚本路径（可选） | `mineru_parse_documents.py`（同目录）|
| `CHROME_DATA_DIR` | Chrome 用户数据目录（用于案例库认证） | 空（必填才能搜索/下载）|
| `CHROME_PROFILE` | Chrome profile 子目录（配合 CHROME_DATA_DIR 使用） | `Default` |

## 人民法院案例库（rmfyalk）功能

### 概述

除法院送达文书外，本 Skill 还支持搜索并下载**人民法院案例库**（rmfyalk.court.gov.cn）的案例。

**注意**：人民法院案例库需要已登录的浏览器会话。执行前请确保已在 Chrome 中登录案例库网站。

### 系统说明

- **网站地址**：`https://rmfyalk.court.gov.cn`
- **认证方式**：OAuth 登录（Cookie/Session），需要浏览器已登录
- **数据格式**：案例正文为 HTML 格式（含基本案情、裁判理由、裁判要旨、关联索引）；PDF 下载通过 API 获取
- **收录规模**：截至 2026 年 4 月，收录约 5377 篇案例

### 搜索参数说明

| 参数 | 含义 | 示例 |
|------|------|------|
| `key=qw` | 全文检索 | `key=qw` |
| `key=ay` | 案由检索 | `key=ay` |
| `isAdvSearch=0` | 非高级检索（普通搜索） | `isAdvSearch=0` |
| `isAdvSearch=1/2` | 高级检索（精确/模糊） | `isAdvSearch=2` |
| `lib=cpwsAl_qb` | 案例库全文检索 | `lib=cpwsAl_qb` |
| `lib=ck` | 参考案例 | `lib=ck` |
| `lib=zdx` | 指导性案例 | `lib=zdx` |

### 搜索结果字段说明

每个搜索结果包含以下字段：

| 字段 | 含义 | 示例 |
|------|------|------|
| `title` | 案例标题 | 扬州四某环保设备有限公司诉... |
| `case_no` | 案号 | （2022）最高法知民终2527号 |
| `court` | 审理法院 | 最高人民法院 |
| `date` | 裁判日期 | 2024.10.23 |
| `category` | 案件类型 | 民事/刑事/行政/执行/国家赔偿 |
| `reason` | 案由 | 发明专利权权属、侵权纠纷 |
| `procedure` | 审理程序 | 一审/二审/再审/其他审理程序 |
| `storage_date` | 入库日期 | 2026.04.13 |
| `id` | 案例 ID（Base64 编码） | lmACF1R47BWcr2N... |
| `lib` | 案例类型 | `ck`=参考案例, `zdx`=指导性案例 |

### 用法

```bash
# 搜索案例（仅搜索，不下载）
python rmfyalk_downloader.py search "<关键词>" --pages 3

# 搜索 + 下载全部结果
python rmfyalk_downloader.py search "<关键词>" --pages 3 --download

# 搜索 + 下载指定序号（第1、3、5条）
python rmfyalk_downloader.py search "<关键词>" --indices "1,3,5"

# 指定输出目录
python rmfyalk_downloader.py search "<关键词>" --output "D:/案例"

# 获取单个案例详情（JSON）
python rmfyalk_downloader.py case "<案例URL或ID>"

# 下载指定案例（通过URL或ID）
python rmfyalk_downloader.py download "<案例ID>" --output "D:/案例"
```

### 输出说明

- **下载文件**：`{案号}.pdf`（如 `（2022）最高法知民终2527号.pdf`）
- **JSON 元数据**：每个案例同时保存同名的 `.json` 文件
- **搜索结果汇总**：所有案例搜索后保存 `搜索结果_{关键词}_{数量}条.json`

### 案例解析字段（JSON）

下载的 JSON 包含完整的案例结构化信息：

| 字段 | 含义 |
|------|------|
| `title` | 完整标题（含副标题） |
| `case_no` | 案号 |
| `court` | 审理法院 |
| `date` | 裁判日期 |
| `category` | 案件类型 |
| `reason` | 案由 |
| `procedure` | 审理程序 |
| `storage_date` | 入库日期 |
| `裁判要旨` | 裁判要点（核心适用规则） |
| `基本案情` | 案件事实摘要 |
| `裁判理由` | 法院论述 |
| `关联索引` | 引用法条及关联案件 |
| `download_url` | PDF 下载直链 |

### 注意事项

1. **认证（关键）**：人民法院案例库需要有效登录会话，否则搜索返回空结果。
   配置方式：将 Chrome 用户数据目录路径设为环境变量，例如：
   ```bash
   # Windows PowerShell
   $env:CHROME_DATA_DIR = "$env:LOCALAPPDATA\Google\Chrome\User Data"
   # 然后运行脚本
   python rmfyalk_downloader.py search "关键词" --pages 3
   ```
   注意：需要 Chrome **完全关闭**（无任何 Chrome 窗口运行）才能使用同一用户数据目录。

2. **翻页**：默认搜索 3 页（约 30 条），如需更多结果增加 `--pages` 参数
3. **下载限制**：指导性案例（lib=zdx）下载方式与参考案例相同
4. **文件命名**：优先使用案号命名；无案号时截取标题前30字
5. **OCR**：案例正文为 HTML，无需 OCR；仅当需要原始 PDF 时下载
6. **无认证时**：如果无法配置 Chrome 目录，可以先在 Chrome 中登录案例库，然后让我（AI助手）通过浏览器自动化工具帮你搜索和下载案例
