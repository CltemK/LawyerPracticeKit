# 法律文件重命名工具 - 规格说明书

## 1. 项目概述

- **项目名称**: LegalFileRenamer (法律文件重命名器)
- **类型**: Windows 桌面应用程序 (WPF/.NET 8)
- **核心功能**: 在Windows资源管理器右键菜单中添加"法律文件重命名"选项，用户可通过友好的对话框对法律文件进行规范化命名
- **目标用户**: 律师、法务人员、行政人员等需要管理大量法律文档的用户

## 2. UI/UX 规格

### 2.1 窗口结构

- **主窗口**: 单个模态对话框 (Modal Dialog)
- **窗口属性**:
  - 固定大小，不可调整
  - 居中显示在父窗口
  - 无最大化/最小化按钮
  - 顶部标题栏带关闭按钮

### 2.2 视觉设计

**颜色方案**:
- 主色: `#1E3A5F` (深蓝色 - 法律/专业感)
- 次色: `#F5F5F5` (浅灰背景)
- 强调色: `#2E7D32` (绿色 - 确认按钮)
- 警告色: `#D32F2F` (红色 - 取消按钮)
- 文字色: `#212121` (深灰)
- 边框色: `#BDBDBD` (中灰)

**字体**:
- 标题: Microsoft YaHei UI, 16px, Bold
- 标签: Microsoft YaHei UI, 13px, Regular
- 输入框: Microsoft YaHei UI, 14px, Regular
- 提示文字: Microsoft YaHei UI, 11px, Regular

**间距系统**:
- 窗口内边距: 24px
- 元素垂直间距: 16px
- 标签与输入框间距: 8px

### 2.3 组件列表

| 组件 | 描述 | 状态 |
|------|------|------|
| 原文件路径显示 | 只读文本框，显示选中文件路径 | 默认 |
| 日期输入框 | 6位数字输入，placeholder显示格式提示 | 默认/聚焦 |
| 文件类型下拉框 | 4个选项：客户、初稿、修订Vx、终稿 | 默认/展开 |
| 文件名称输入框 | 原文件名显示为默认值 | 默认/聚焦 |
| 版本号输入框 | 带版本号格式提示 | 默认/聚焦 |
| 确认按钮 | 绿色，执行重命名 | 默认/悬停/点击 |
| 取消按钮 | 红色，关闭对话框 | 默认/悬停/点击 |
| 预览标签 | 实时显示新文件名预览 | 动态更新 |

## 3. 功能规格

### 3.1 核心功能流程

1. 用户在资源管理器中右键点击文件
2. 选择"法律文件重命名"菜单项
3. 应用程序启动，显示重命名对话框
4. 用户在4个框内分别输入/选择：
   - **日期**: 默认当日，格式 YYYYMMDD (如 20260418)
   - **文件类型**: 下拉选择
     - 客户: `【客户】`
     - 初稿: `【初稿】`
     - 修订Vx: `【修订V1】` (用户需输入版本)
     - 终稿: `【终稿】`
   - **文件名称**: 默认为原文件名
   - **版本号**: 格式 V1、V2... 或 V1.1
5. 实时预览新文件名
6. 点击确认执行重命名
7. 关闭对话框

### 3.2 文件名格式

```
{日期}-{文件类型}-{文件名称}-{版本号}.{扩展名}
```

**示例**:
- `20260418-【客户】-采购合同-V1.pdf`
- `20260418-【初稿】-劳动合同修订V2-V1.1.docx`
- `20260418-【终稿】-协议-V2.xlsx`

### 3.3 数据校验

- 日期: 必须为6位数字，前4位年(1900-2100)，中间2位月(01-12)，后2位日(01-31)
- 文件类型: 必选
- 文件名称: 必填，不可为空或仅空格
- 版本号: 必填，格式 V开头 + 数字，可带小数点

### 3.4 右键菜单注册

- 注册位置: `HKEY_CURRENT_USER\Software\Classes\*\shell\LegalFileRenamer`
- 支持文件类型: 所有文件 (`*`)
- 显示名称: "法律文件重命名"

## 4. 技术架构

- **框架**: .NET 8 + WPF
- **构建输出**: 单个可执行文件 + 安装脚本
- **注册方式**: Registry写入 (HKCU，用户级无需管理员)
